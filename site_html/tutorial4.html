If we click the button, it will show alert message from server.

index.html

<html>
	<head>
		<script src="jquery.js"></script>
		<script>
			function on_click() {
				$.post('jquery_post', {name : 'abc', num : 123}, 
					function(data, status) { alert(data); }
				);
			}
		</script>
	</head>
	<body>
	<button onclick='on_click()'>post with jquery</button><br>
	</body>
</html>

C++ side Programming

	void process() {
		if(requested_document_ == "jquery_post") 
				content_ = "response from server";
		}
Yes, we don't have jquery_post file in our web directory. So content_ is empty this time. If there is any file in our html directory, the CWFS library will fill the content_ variable with it. If it cannot find one, it will leave content_ empty. And, requested document_ can be anything. Not like jsp, asp, it can be html file or sometimes non-existing file. We can use this just as a variable. In this case, to verify a jquery request.
